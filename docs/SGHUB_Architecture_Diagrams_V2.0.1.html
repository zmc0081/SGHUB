FIG-01 · SGHUB V2.0.1 整体分层架构
三层架构:UI Layer (WebView) → Core Layer (Rust) → OS Layer · 全部运行在用户本地
所有组件运行在用户本地单进程中,无需云端服务器。UI 层通过 Tauri IPC 与 Core 层通信,Core 层通过 Tauri Plugin 调用 OS 原生能力。

FIG-04 · 数据模型 ER 图
SQLite 单文件数据库 · 13 个核心表 (V2.0.1 新增 chat_sessions / chat_messages / chat_attachments) · ~/.sghub/data/sghub.db
所有表存储在单个 SQLite 文件中。API Key 不在数据库中存储,ModelConfig.keychain_ref 仅为 OS Keychain 的引用标识符。V2.0.1 新增 chat_* 系列表用于存储自由对话会话。
FIG-05 · Chat 对话数据流 V2.0.1 NEW
参考 Claude.ai 风格的自由对话界面 · 复用 ai_client + skill_engine · 支持附件 / Skill / 文献引用
Chat 与 AI 解析共享底层基础设施(ai_client / keychain / skill_engine),通过会话粒度的上下文累积实现自由对话。前端流式渲染通过 Tauri Event "chat:token" 实现,延迟 < 100ms。
